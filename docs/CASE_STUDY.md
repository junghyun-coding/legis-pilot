## 1. 문제

국민신문고·입법제안·민원으로 정책 아이디어는 쏟아지지만, *"이 제안이 상위법과 충돌하는가,
위헌 소지는 없는가, 시행령만으로 가능한가 아니면 법률 개정이 필요한가"* 라는 **1차 법적
타당성 검토는 여전히 사람이 수작업**으로 한다.

그렇다고 LLM에 그냥 물으면 — **존재하지 않는 법조문과 판례를 그럴듯하게 지어낸다.**
행정 의사결정에서는 치명적인 결함이다.

> **핵심 질문:** AI의 분석력을 쓰되, 근거의 실재(實在)를 어떻게 보장할 것인가?

## 2. 해결

분석에 쓰는 모든 근거를 **국가법령정보 Open API로 실재가 확인된 데이터로 못박고**, AI는
그 위에서만 판단하게 했다. 4단계 파이프라인으로 동작한다.

1. 일상어 제안 → 법령 검색어·키워드 추출
2. 현행법령·법령해석례·헌재결정례·자치법규 **4종 병렬 실시간 조회**
3. 최상위 법령의 **실제 조문 본문을 읽고(RAG)**, 4개 판단축을 **각각 o3로 병렬 심층분석** → 종합
4. 타당성 판정 + 입법형식 판단 + 부처 협의용 **1페이지 리포트** 자동 생성

## 3. 핵심 기술 결정

**① 할루시네이션을 "설계"로 차단**
AI에게 법령을 기억해서 말하라고 하지 않는다. API로 가져온 실존 목록과 조문 본문만
컨텍스트로 주고 *"여기 없는 건 지어내지 말라"* 고 강제한다. 인용된 법령·판례는 전부 클릭하면
국가법령정보 원문으로 연결된다.

**② 단일 호출 → 4축 멀티에이전트**
처음엔 o3 한 번으로 8개 필드를 모두 뽑았다. 각 판단축이 얕아지는 한계가 보여,
법적정합성·유권해석 부합·사법안전성·시행령 실현가능성을 **전문 분석관 4명으로 분리**해
`reasoning_effort: high`로 병렬 분석하고, 수석검토관이 종합하도록 재설계했다. 점수는 각
전문가가 산정하고 종합은 서술만 통합한다. 병렬이라 지연은 단일 대비 크게 늘지 않는다.

**③ 검색 관련성 게이트**
법령해석례 검색이 구체어로 0건이면 `'피해자'` 같은 일반어로 폴백되며 무관한 해석례(가정폭력·
참사 등)가 쏟아지는 문제가 있었다. **구체 주제어 기반 관련성 필터**로 노이즈를 걸러내고,
진짜 관련 결과가 없으면 가짜를 만드는 대신 **섹션을 비웠다(정직성 우선).**

**④ 절대 비지 않는 폴백 체인**
API 키 없음/장애 → 실존 법령명 시드, LLM 실패 → 경량 모델 → 목업. 어떤 상황에서도 데모가
끊기지 않도록 설계했다.

## 4. 결과

입력 *"보이스피싱 피해자 즉시 계좌 동결 권한 부여"* 에 대해:

- **27.5초** 만에 1페이지 리포트 생성 (o3 5콜 + 추출 1콜)
- **정비 권고 / 타당성 80점** (법적정합성 82 · 유권해석 82 · 사법안전 75 · 실현가능성 85)
- 근거: *"전기통신금융사기특별법 제3조제1항이 이미 부여한 피해자 신청권의 전자적 확대에
  불과해 법체계와 부합"* — **실제 조문을 직접 인용**한 분석

데이터 출처 배지로 *국가법령정보 API 실연동* 여부를 리포트에 명시하고, `리포트 PDF 저장`으로
부처 협의용 산출물을 바로 뽑을 수 있다.

## 5. 한계와 배운 점

- 판정 점수는 LLM 추정값이라 **정답셋 기반 정확도 평가는 아직 못 했다** — 다음 우선순위.
- "근거의 실재를 보장한다"와 "조문 내용을 한 줄씩 검증한다"는 다른 문제다. 전자는 달성했고
  후자(복수 법령 교차 대조)는 로드맵에 남겼다.
- **가장 큰 배움:** LLM 제품의 신뢰성은 모델 성능이 아니라 *"근거를 어떻게 고정하고, 모를
  때 어떻게 정직하게 비울 것인가"* 라는 **설계**에서 나온다.
